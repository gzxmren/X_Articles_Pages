1⃣：什么为EA量化策略？
EA 量化策略（Expert Advisor，专家顾问），是一种自动执行的量化交易程序，主要运行在 MT4/MT5 平台，多用于外汇、黄金、期货、数字货币等市场
一、核心含义
- EA：一段自动交易代码（MQL4/MQL5 编写）
- 量化策略：把买卖、止损、仓位、风控写成明确、可量化的规则
- 合起来：人定规则 → 写成代码 → 电脑自动执行，全程无需人工盯盘、手动下单
二、它在做什么（一句话）
7×24 小时盯盘 → 识别信号 → 自动开仓/平仓/止盈止损 → 严格按规则、不带情绪执行
三、常见 EA 策略类型（不是我）
1. 趋势跟踪（最主流）
- 均线金叉买、死叉卖；突破通道做多/空
- 适合：大涨/大跌的单边行情
2. 网格/震荡策略
- 区间内：跌了买、涨了卖
- 适合：横盘震荡市
3. 马丁策略（高风险）
- 亏损后加倍加仓，赚一次回本
- 风险：单边大行情易爆仓
4. 对冲/套利
- 多空同时持仓、降低波动
5. 高频/剥头皮
- 超短线、赚点差、毫秒级下单
四、优点与风险
✅ 优点
- 无情绪、严格执行
- 24 小时运行、毫秒响应
- 可历史回测、多品种同时跑
⚠️ 风险
- 不是“稳赚”：策略本身决定盈亏
- 过度拟合（只适配历史、实盘失效）
- 网络/平台故障、滑点、黑天鹅风险
- 马丁、高杠杆策略极易爆仓
一句话总结：
EA 是工具，不是印钞机。关键是策略逻辑 + 严格风控

自动交易 Bot 使用前说明书（初步版）---（免责声明）
适用文件：北境量化策略V5-1（4.13）.PY
一、这是什么
这是一个用于自动扫描、自动判定、自动执行、自动复检的交易脚本。
它主要面向两个永续合约网页平台的联动操作，程序会通过浏览器页面读取信息、做策略判断，并在满足条件时自动下单或平仓。
这份说明书只讲使用前需要知道的事，不讲内部代码和函数细节。
二、适合谁使用
这个程序更适合下面这类用户：
已经长期在用对应交易平台
知道自己的策略逻辑和风险承受范围
能看懂基本的持仓、名义价值、盈亏、方向、多空这些内容
能接受 bot 是“按设定自动执行”，不是每一步都人工确认
如果你还不熟悉平台页面、仓位结构、自动下单风险，那建议先不要直接实盘跑。
三、程序大概会做什么
程序启动后，通常会做这些事：
打开或接管浏览器页面
读取平台页面信息
扫描币种、仓位、盈亏、指标等
按预设策略判断是否需要执行动作
自动下单 / 平仓 / 加仓 / 复检
输出日志、摘要、面板结果
你可以把它理解成：
“会自己看页面、自己判断、自己点按钮的交易执行助手。”
四、运行环境要求
这是一个 Windows 环境下运行的 Python 脚本，并且依赖浏览器页面自动化。
建议环境如下：
1）操作系统
Windows 10 或 Windows 11
建议使用稳定、常用的桌面环境
不建议远程环境频繁断连、锁屏、休眠
2）Python
需要本机已安装 Python
建议使用较新的 Python 3 环境
要保证命令行里可以直接运行 .py 文件
3）浏览器
需要本机可正常打开浏览器
浏览器页面必须能正常访问目标交易平台
运行期间不要随意手动关闭 bot 正在使用的页面
4）网络
网络要稳定
页面必须能正常打开、正常加载、正常切币、正常读取内容
如果平台本身打不开、卡死、频繁掉线，bot 也没法正常工作
5）电脑状态
不要让电脑自动休眠
不要让系统频繁弹权限框、更新框、杀毒拦截框
不建议同时开太多高占用软件，避免浏览器卡顿
五、启动前需要准备哪些东西
在正式运行前，你至少要准备好下面这些：
1）交易平台账号
你需要已经有对应平台的可用账号，并且：
能正常登录
能正常看到持仓
能正常下单
页面不是异常状态
2）浏览器登录状态
运行前最好先确认：
平台已经登录
没有卡在登录页
没有卡在人机验证页
没有弹出额外确认页
页面不是白屏、空白页、崩溃页
3）策略预期
你要明确知道这个 bot 跑的是你当前认可的那套逻辑。
至少要清楚：
它会不会自动开仓
它会不会自动平仓
它会不会自动加仓
它是跑哪些币
它是单平台还是双平台联动
也就是说，先确认“它会怎么做”，再让它运行。
4）资金与仓位准备
正式启动前，建议先检查：
账户余额是否足够
当前仓位是否符合你的预期
有没有遗留旧仓位
有没有你不想让 bot 动的仓位
平台最小下单限制你是否清楚
5）目录与权限
脚本所在目录要具备正常读写权限。
因为程序通常会写出：
日志
面板输出
摘要
复检结果
其它运行记录
如果目录没有权限，后面排查问题会很麻烦。
六、第一次运行前，建议先做的检查
第一次用，建议先按这个顺序过一遍。
1）先手动打开平台页面
确认两个平台页面都能正常打开，不是空白，不是报错，不是卡死。
2）先手动确认页面可操作
随便点一下页面里的常规区域，确认页面不是假死状态。
3）先确认账户信息正常
能看到持仓、价格、按钮、方向这些关键内容。
4）先确认脚本路径干净
不要混着太多旧版本脚本，避免你自己运行错文件。
5）先看清楚本次要运行的是哪一个文件
避免“以为跑的是新版本，实际跑的是老版本”。
七、启动后你应该怎么使用
这个 bot 的正常使用方式，不是一直去手动干预它。
更合理的方式是：
1）启动后先观察
不要一启动就乱点页面。
先看它能不能：
正常接管页面
正常扫描
正常输出日志
正常识别页面
2）优先看日志和摘要
判断程序是否正常，最关键不是“它有没有动”，而是看：
它有没有开始扫描
它有没有识别到页面
它有没有给出明确原因
它有没有执行成功或失败说明
3）不要和 bot 抢页面
如果 bot 正在操作某个平台页面，尽量不要同时手动乱切、乱点、乱改页面内容。
否则容易出现：
读错内容
点错按钮
复检失真
状态判断混乱
4）发现异常先看是不是页面问题
很多“程序异常”，本质其实是页面异常，比如：
平台网页没加载出来
标签页空白
页面卡住
平台自己报错
登录状态丢了
先分清楚是“程序错了”，还是“网页本身就不正常”。
八、运行期间的注意事项
这是最重要的一部分。
1）不要频繁手动关闭页面
如果你手动把 bot 正在使用的关键页面关掉，程序后面可能会重新建页、重连、恢复，甚至误判。
2）不要临时换网络环境
例如突然切代理、断网、切 Wi-Fi，这会导致页面状态大幅波动。
3）不要边运行边随便改脚本
脚本运行时不要一边运行一边乱覆盖文件，不然你自己都说不清当前到底跑的是哪版。
4）不要把它当成“绝对不会错”
它本质上还是自动化程序，不是神。
页面变了、平台卡了、网络波动了，都可能影响执行结果。
5）不要无人值守太久
尤其是新版本、刚修完的版本，建议先有人盯着跑一段时间，看清楚行为是否符合预期。
九、建议的使用顺序
如果你是第一次跑，建议按这个顺序：
第一步：空载观察
先启动脚本，只观察它能否正常接管页面、输出日志、识别环境。
第二步：轻仓验证
确认页面接管正常后，再用较轻的真实仓位环境验证。
第三步：观察复检和日志
重点看它执行后有没有正常记录、正常复检、正常输出原因。
第四步：确认稳定后再长期运行
不要一上来就满负荷实盘长期挂机。
十、常见误区
误区 1：页面能看到，就等于 bot 一定能正常识别
不一定。
人眼看着正常，不代表页面元素对自动化来说就是稳定可读的。
误区 2：程序没下单，就是程序坏了
不一定。
也可能是当前轮没有满足策略条件，或者被规则拦住了。
误区 3：只要报错就是脚本逻辑错
不一定。
有些报错本质是网页加载失败、平台接口慢、浏览器异常、页面空白。
误区 4：运行时手动乱干预没事
不对。
很多“莫名其妙的异常”，就是人和 bot 同时抢页面造成的。
十一、使用前最后确认清单
正式启动前，建议你自己问自己这几句：
我打开的是不是对的脚本文件？
两个平台页面是不是都正常？
账号是不是已经登录？
当前仓位是不是我认可的状态？
网络是不是稳定？
我知不知道这版 bot 会自动做哪些动作？
我这次启动后，是否能盯一段时间看运行情况？
只要这些没确认清楚，就不建议直接开跑。
十二、一句话总结
这份 bot 的正确使用思路，不是“点开就不管了”，而是：
先确认环境正常，再启动；先观察行为正常，再逐步放开
