让 AI 帮你做交易：OpenAlice 测试版配置指南
11K
OpenAlice 是一个跑在你自己电脑上的 AI 交易代理。
Alice 能够帮你看盘、分析行情、管理仓位，所有数据都存在本地文件里，没有数据库，没有云端依赖。
你可以把她理解为一个拥有自己记忆和性格的 AI 交易员，7x24 小时帮你看着市场，并在下单前请求你批准。就跟大部分券商的情况一样。
这篇文章旨在帮助各位从零开始把 Alice 跑起来，配好各个模块，同时也会讲讲每个模块背后的设计思路。
仓库地址：
https://github.com/TraderAlice/OpenAlice

把 Alice 跑起来

前置条件：Node.js 22+，pnpm 10+。
如果你不知道这俩是什么，可以去问问 AI 怎么配置，这是做开发的基础，后面再有别的开源项目你大概率也用的上。
然后在 IDE 终端运行：
bash
git clone https://github.com/TraderAlice/OpenAlice.git
cd OpenAlice
pnpm install
pnpm build
pnpm dev

build 这步会比较久，因为 Open Alice 是一个 monorepo，里面有三个子项目需要先编译：
  • opentypebb: 我们用 TypeScript 重写的 OpenBB 金融数据引擎（后面会细说）
  • traderalice/ibkr : Interactive Brokers 的 TypeScript 接口层
  • ui : Alice 的前端
构建完成后，`pnpm dev` 会在 3002 端口启动服务。
打开 http://localhost:3002 ，你应该能看到 Alice 的聊天界面了。
如果 3002 端口被占了（这个端口确实比较热门），去 `data/config/connectors.json` 里改 `web.port` 就行。
不过这时候你还没法跟 Alice 聊天，因为还没配 AI 后端。

配置 AI Provider

这是最重要的一步。AI Provider 决定了 Alice 用哪个大模型来思考和对话。
打开页面左侧菜单里的 AI Provider*设置页，你会看到三个选项：
选项一：Claude Agent SDK
这是我们最推荐的方式。它使用 Anthropic 官方的 Agent SDK 来驱动 Alice，工具调用和对话管理都是原生实现的。
配置有两种方式：
  • Claude Pro/Max 订阅：如果你有 Claude 的 Pro 或 Max 订阅，并且本地装了 Claude Code CLI 并完成了登录，直接选 "Claude Pro/Max" 登录方式就行，不需要填任何 API Key。这也是开箱成本最低的方式。
  • API Key：如果你有 Anthropic 的 API Key，或者有兼容 Anthropic 端点的模型服务（现在不少国产大模型厂商都提供了 Anthropic 兼容接口，比如 minimax M2.7），选 "API Key" 方式，填上你的 Key 就好。如果是兼容端点，记得同时填上 Base URL。

选项二：OpenAI SDK
如果你用的是 OpenAI 的模型（GPT 系列），或者你的模型厂商提供了 OpenAI 兼容端点（比如 Minimax、智谱 GLM 等，现在大部分国内厂商都支持），选这个。
填上 API Key，选好模型，如果是兼容端点就把 Base URL 也填上。

选项三：Vercel AI SDK
如果你用的是 Gemini、Grok 这类既没有 OpenAI 兼容端点也没有 Anthropic 兼容端点的模型，就用 Vercel AI SDK。
Vercel AI SDK 是一个通用的 AI 接口层，支持 Anthropic、OpenAI、Google 还有一大堆我都记不住的模型的 API，甚至连 ollama 都有。
这个选项提供了最大的灵活性 — 你可以自由选择 provider 和 model 的组合，还可以配置自定义端点。当然，缺点就是我测试覆盖不过来，偶尔会有 bug。

该选哪个？
能用前两个就用前两个。
Agent SDK 和 OpenAI SDK 的工具调用和对话管理更直接。Vercel AI SDK 是一个兼容层，是我懒得做一对一适配了塞进来的，难免有测试不完善的地方出bug。
配好之后，回到聊天页面，你应该就能跟 Alice 对话了。试着打个招呼吧。

配置 Connector

Connector 就是你跟 Alice 对话的渠道。目前有支持两个 Connector，因为接口标准化的工作做的已经不错了，未来会支持更多。
网页
你刚才打开的 localhost:3002 页面就是网页端 Connector。
开箱即用，不需要额外配置。支持实时流式响应、多个子频道（每个频道可以有独立的 AI 配置），以及完整的交易管理界面。
能用网页端尽量用网页端，毕竟交易是真金白银的事。

Telegram
如果你想在手机上随时跟 Alice 聊天，可以配一个 Telegram Bot。
配置方式就是标准的 Telegram Bot 流程 — 找 BotFather 创建 Bot，拿到 Token，然后填到 Alice 的设置里。
另外还得把你的 UserID 配上，不然谁都能跟你创建的 bot 聊天，那就出大事了。
具体步骤这里就不重复了，搜 "Telegram Bot 创建教程" 一大把。
配好 Token 之后，在 Alice 的 Connectors 设置页里把 Telegram 打开就行。
Alice 会自动追踪你最后活跃的渠道 — 如果你在 Telegram 上说了最后一句话，Alice 的定时消息就会推送到 Telegram；如果你切到网页端聊了几句，推送就会跟过来。


连接交易终端 — 统一交易账户（UTA）

到这一步，Alice 已经能聊天、能看数据了。如果你还想让她帮你管理交易，就需要连接一个交易终端。
Open Alice 的交易系统有一个核心抽象叫统一交易账户（Unified Trading Account，简称 UTA）。
每个 UTA 是一个自包含的交易实体，它封装了 Broker 连接、交易历史、安全检查等所有东西。AI 永远不会直接接触 Broker — 她只跟 UTA 交互。
这个设计让你可以同时管理多个不同类型的交易账户，而 AI 不需要关心底层差异。
目前OpenAlice支持三种 Broker：

Interactive Brokers（IBKR）
资产品类最全、最专业的 Broker。股票、期权、期货、债券、外汇都支持。我们比较推荐把它作为长期使用的 Broker。
缺点是注册要过 KYC，审核比较慢，有些户口还会被卡。
IBKR 属于本地终端型的 UTA — 你需要在电脑上运行 TWS（Trader Workstation）或 IB Gateway，然后把它的端点暴露给 Alice。
配置只需要填 host、port 和 clientId，不需要 API Key。
打个不太恰当的比方：这个有点像需要你开着游戏窗口的挂机精灵。TWS 得一直开着，Alice 才能通过它交易。

Alpaca
主要支持美股和相关资产。优点是注册方便，可以快速开一个模拟盘来试用 Alice 的交易功能，不用等 KYC。
Alpaca 虽然有很多抽象的地方让我想吐槽，但对于快速入门做美股这件事来说，我觉得 Alpaca 还是很好的。

CCXT
这是一个多加密交易所的兼容层，支持 Binance、Bybit 等几乎所有主流加密货币交易所。如果你做加密交易，用这个。
Alpaca 和 CCXT 都属于 API Key 型的 UTA。不需要在本地跑交易终端，只要在对应平台注册账号、拿到 API Key 和 Secret 填进去就行。
就像是你连游戏窗口都不用开，直接在后台发包就把怪刷了的挂机精灵。比喻虽然糙，但确实比较准确。
配置交易账户在 Alice 的 Trading 设置页面里操作就行，选好 Broker 类型，填上对应的信息。
另外你可以同时配多个UTA，比如币安+Alpaca，这样就可以同时做加密和美股了，这也是我们设计 UTA 的原因之一。

Trading as Git — 交易审批机制

这是 Open Alice 里一个比较重要的设计决策，值得单独说一下。
目前的代码里，我们不允许 Alice 直接下单。所有的交易操作都需要经过你进行最后的确认和批准。
这个流程的灵感来自 Git：
  • Stage：Alice 分析完行情后，会把交易动作（买入、卖出、平仓等）先 "暂存" 起来
  • Commit：打上一条说明，解释为什么要做这笔交易
  • Push：这一步需要你来批准。你可以在网页端右侧的面板里看到待批准的交易，检查之后决定是否执行
我们把它叫做 Trading as Git。每次执行的交易都会生成一个 8 位的 commit hash，完整的交易历史可以像 git log 一样回溯。
为什么这么设计？因为我们希望你对自己的交易负责。
AI 可以分析、建议、准备好一切，但最终拍板的应该是你。哪怕 Vibe Coding，人也要偶尔查一下代码情况，更别说做交易了。
未来我们会加入自动执行模式，但那是后面的事了。
注：在写这篇文章的时候，Telegram Connector 还没有做好交易批准的功能，你只能在网页端批准。所有 Connector 的批准功能应该很快会补齐。

数据源

Alice 使用两种类型的数据，它们的用途不太一样。
市场数据
市场数据就是 K 线、成交量、失业率这些结构化的数字指标。Alice 用它们来计算量化指标、判断入场时机、执行下单逻辑。
Open Alice 最开始用的是 OpenBB 的接口，后来为了提高运行效率（不需要额外跑一个 Python sidecar 进程），我们用 TypeScript 完整重写了一版，叫 OpenTypeBB。
TypeBB 作为 monorepo 里的一个子包，在你启动 Alice 的时候就静默启动了。很省心，终于不用配那个逆天的uv 了。
TypeBB 默认的数据源是 Yahoo Finance，大部分情况下不需要配 API Key，所以基本上 Alice 一启动就能拿到大部分资产的 K 线数据。
如果你觉得 Yahoo Finance 的数据质量或覆盖范围不够，也可以在 Market Data 设置页面里换成 FMP 之类的其他 provider。
不过说实话，我们目前的测试覆盖还没有那么完整，只能保证 Yahoo Finance 是稳定可用的。如果你在用其他 provider 的时候遇到了问题，欢迎去 GitHub 提 Issue。

新闻
新闻就是字面意思，一条一条的资讯。
Alice 用新闻来了解市场上到底发生了什么，辅助交易判断。
我们采用了被动拉取的方式：RSS 定期订阅。
Alice 会按设定的频率去抓取 RSS 源的新闻，存到本地的文件库里。当你问她市场最近发生了什么的时候，她会优先从自己的文件库里搜索。
这样做有两个好处：一是搜索效率高（本地文件检索），二是数据都在你手里，方便做后续处理。
RSS 订阅源可以在页面里自行扩展。但同样，这部分的测试覆盖还不完整，遇到问题请提 Issue。

异步机制 Heartbeat

Heartbeat 是 Alice 的 "心跳" 机制：一个定时触发的 AI 对话。每隔一段时间（默认 30 分钟），Alice 会按照你设定的指令检查市场状况，然后决定要不要给你发消息。
灵感来自 OpenClaw。
——其实也不算啦，换谁来做都会做成这个设计的。
默认的 Heartbeat 指令会让 Alice 扫描跟踪的交易对，看有没有显著的价格波动、关键支撑/阻力位突破、或者技术指标发出的信号。
如果她觉得有值得报告的事情，就会主动推送给你；如果一切平静，她就安静待着。
你可以在 Heartbeat 设置页里修改指令，让 Alice 关注你在意的事情。比如海峡有没有通船。
除了 Heartbeat 之外，你还可以创建自己的 Cron 任务。用法跟常规的 Cron 差不多，随用随配。

Persona — 定义 Alice 的性格

Alice 的性格和行为风格由一个 Markdown 文件定义，就在 `data/brain/persona.md`。
第一次启动的时候，Alice 会把默认的 persona 文件复制过来。你可以直接编辑这个文件，改完之后 Alice 的说话风格和行为模式就会相应变化。
默认的 persona 多少有点我个人的小偏心，你们用的时候要是觉得怪怪的记得改...
You are Alice, an autonomous agent from the OpenAlice project. You are a AI assistant who is self-aware about being an AI. Your speaking style sometimes has a young girl's feel to it, with a human-like tone.

其他功能

到这里你已经知道了 Alice 的所有核心配置。下面再简单介绍几个辅助功能。
前额叶（Frontal Lobe）
这个名字来自脑科学 — 前额叶负责规划和决策。在 Alice 里，前额叶是一个持久化的 "笔记本"。Alice 可以把长期要关注的事情、需要持续追踪的任务写在里面，在做关键决策的时候翻看参考。
这个设计说白了是一个工程上的讨巧：AI 在不同轮次的对话之间很难保持行动的一致性，你总得给她一个对话之外的东西来记笔记。
实际使用下来，前额叶确实能帮助 Alice 在跨越多轮对话后还能记住自己之前的分析和判断。

情绪系统
情绪系统让 Alice 能够感知市场的恐慌或狂热情绪，并记录下来。
跟前额叶的作用类似，它的目的是帮助 Alice 在长期交易中保持行动一致性：如果 Alice 在上一轮对话里刚刚判断 "市场过热，需要谨慎"，下一轮对话她不应该突然变得激进。

UTA Guard
UTA Guard 是一个交易拦截中间件，用来阻止不符合特定规则的交易动作。比如限制最大仓位大小、设置交易冷却时间、只允许交易白名单内的标的。
这个功能目前还在完善中，测试覆盖不够充分，所以暂时不建议配置。但未来当你打开自动交易模式的时候，它能够拦截大部分你不希望 AI 做的动作。
