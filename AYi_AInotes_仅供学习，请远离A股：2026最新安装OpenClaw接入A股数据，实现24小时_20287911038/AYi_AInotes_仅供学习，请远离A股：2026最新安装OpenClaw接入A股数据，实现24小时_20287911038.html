六、常见问题排查

（一）数据接入失败
  1. 现象：输入分析指令后，系统提示“数据获取失败”或“API Key无效”；
  2. 排查步骤： 验证API Key正确性：重新检查配置的QVeris/Tushare密钥，确保无空格、换行； 测试网络连通性：
# 测试QVeris接口连通性
curl https://api.qveris.ai/v1/tools/stock/a股实时行情
检查技能状态： openclaw skills list | grep qveris ，确保状态为“ready”； 重启服务： openclaw gateway restart 。
（二）定时任务未执行
  1. 现象：创建定时任务后，未按预期生成分析结果；
  2. 排查步骤： 检查Cron表达式格式：确保时间配置正确（如股市交易日为周一至周五）； 查看任务日志： openclaw logs --service cron --name pre-market-analysis 验证服务状态： openclaw gateway status ，确保服务正常运行。
（三）多平台联动无响应
  1. 现象：在飞书/Telegram发送指令后，机器人未回复；
  2. 排查步骤： 检查渠道配置： openclaw channels list ，确保状态为“connected”； 验证接口权限：确认飞书/Telegram机器人已开启“消息接收”权限； 查看渠道日志： openclaw logs --service channels --channel feishu
（四）数据更新延迟
  1. 现象：获取的股票数据与行情软件存在较大差异；
  2. 排查步骤： 更换服务器地域：选择中国香港/上海等靠近数据源的地域； 调整数据刷新频率：在指令中明确“获取实时数据”（如“查询A股实时涨幅榜，数据刷新频率1分钟”）； 切换数据接口：尝试用Tushare替代QVeris，对比数据差异。

  1. 密钥保护：QVeris/Tushare API Key是数据访问的核心凭证，切勿泄露给他人，定期更换密钥；
  2. 数据合规：仅用于个人投资分析，不得将数据用于商业用途或非法传播；
  3. 风险提示：AI分析结果仅为参考，不构成投资建议，决策需结合自身风险承受能力；
