写在前面：这篇文章能给你什么

2025 年，OpenAI 官方发布了一篇文章，题目是《Codex 在工作中的十大使用场景》。
这个举动本身就是一个信号：一个曾经被定义为"给程序员用的 AI 编程工具"，开始正式对非技术人员喊话了。
OpenAI 列出的十大场景里，排名第一的是"首席参谋"——Codex 帮你整理消息、管理日历、追踪行动项。这不是一个编程任务，这是知识工作者多年来一直试图用 Zapier、Notion 和各种 AI 助手去自动化但始终没做好的那类工作。
这就是本文要讲的事：不是理论，不是观点，是打工人可以直接复制操作的真实场景，每个场景配上我测试过的 Prompt 模板，你读完可以立刻去试。
在开始之前，先说三件事：
第一，Codex 是什么。 简单说：它是 OpenAI 推出的 AI Agent，基于对自然语言和源代码都做过训练的模型，对结构化指令的理解和结构化输出的生成，比普通 AI 助手更精准——它理解逻辑、序列和数据里的规律。对你来说，这意味着它处理"有规则、有结构、有重复性"的任务，比大多数工具更可靠。
第二，你需要什么权限。 2025 年新版 Codex 已包含在 ChatGPT Plus、Pro 和 Team 订阅中，无需额外付费。
第三，Codex 能力的两个层次。 本文分两个层次展开：前五个场景是"帮你想、帮你写"——你提供素材，它生成内容，你来执行；场景六是"帮你做"——Codex 直接操控电脑、在后台定时自动运行任务，你只需要审核结果。前五个场景是基础，场景六是进阶，建议按顺序读。

场景一：周报、月报、汇报文档——从"每周两小时"到"自动生成等你审"

每周五下午，你坐在电脑前，开始做同一件事：
打开飞书或钉钉，把这周的消息记录翻一遍；打开任务管理工具，把完成的项目截图或粘贴出来；打开上周的周报，参照格式重新填；填完检查一遍用词是否得当；发给领导。
整个过程，快的话一个小时，慢的话两个小时。格式几乎不变，结构几乎不变，需要的信息全部散落在你已经有的地方。
这件事有三种解法，门槛依次升高，省力程度依次增加。
手动喂料版——把素材扔进去，拿回初稿
适合：你愿意花五分钟整理一下原始素材，然后让 Codex 帮你写成结构化的周报。
基础版——适合日常周报：
你是一位职场写作助手，擅长把零散的工作记录整理成清晰的周报。

我的工作记录如下（格式不限，可能包括聊天记录、任务清单、会议纪要片段）：

【粘贴你本周的原始记录】

请按照以下结构生成一份周报初稿：
1. 本周完成事项（分条列出，每条一句话，突出结果而非过程）
2. 进行中的事项（列出名称 + 当前进展 + 下一步行动）
3. 下周计划（列出 3-5 条，优先级从高到低）
4. 需要支持或决策的事项（如有）

语气要求：专业但不过于正式，用词简洁，避免空话套话。
字数控制：每个板块不超过 200 字。
进阶版——适合向上汇报：
你是一位职场写作助手，帮我把周报改写成适合汇报给部门负责人的版本。

原始周报如下：
【粘贴基础版生成的周报】

改写要求：
- 每条内容要体现"业务价值"或"对团队目标的贡献"，不要只列动作
- 把结果数据或可量化的进展放在每条的开头
- 删掉所有"正在推进中""持续跟进"等无实质内容的表述
- 结尾加一句话总结本周的核心成果

目标读者：对细节不感兴趣、只关心进展和风险的部门负责人
月报版——适合月底汇总：
我有以下四周的周报，请帮我整理成一份月报。

第二周周报：【粘贴】
第三周周报：【粘贴】
第四周周报：【粘贴】

月报结构：
1. 本月核心成果（3-5 条，每条强调结果和影响）
2. 主要项目进展（按项目分类汇总）
3. 遇到的挑战和解决方式
4. 下月重点计划

特别说明：把四周里重复出现的事项合并，避免内容重叠。月报语气比周报更正式。
文件直投版——直接出稿（零门槛，一步到位）
不想自己整理原始素材？直接把你有的东西全部丢给 Codex——会议纪要 PDF、任务清单 Excel、聊天记录文本文件——让它读完直接出初稿。

2. 进行中的事项（名称 + 当前进展 + 下一步行动）

适合：记录分散在多个文件里、懒得手动整理素材、或原始记录已经是文件形式（会议纪要、任务表）的情况。
挂机自动版——Automation 定时自动生成（一次配置，长期省力）
最省力的方式：你什么都不用做，Codex 在固定时间自动读取你的日历、邮件、Notion 进度，生成周报初稿推送给你，你打开看一眼、改两句、发出去。
OpenAI 官方给出的旗舰 Automation 场景正是这个：设置一个工作日心跳任务，每天早上 8:30 开始。读取当天日历、过去 24 小时未读的 Slack 消息、未读的 Gmail，以及你的待跟进事项文档。生成一份简报，包括当日优先事项、会议准备需求、需要回复的消息、待决策事项和值得关注的信息，然后每小时检查一次直到下班。
周报自动生成 Prompt（在 Automations 面板设置）：
每周五下午 17:00 自动执行：

1. 读取本周（周一到周五）的 Google Calendar，列出所有完成的会议和事项
2. 读取 Notion 的项目看板，提取本周有更新的任务状态
3. 读取本周收到和发出的 Gmail，找出与工作推进相关的重要邮件
4. 读取我的 Google Doc「本周工作记录」（如有内容）

把以上信息整合，生成一份周报初稿：
1. 本周完成事项（突出结果，不超过 5 条）
2. 进行中的事项（名称 + 当前进展 + 下一步行动）
3. 下周计划（3-5 条）
4. 需要支持的事项（如有）

生成后发送到我的 Gmail 草稿箱，主题格式：「周报初稿」YYYY年第X周。
不要直接发送，等我审阅后手动发出。
所需插件：Google Calendar、Gmail、Notion、Google Drive。
设置方法：Codex App 左侧边栏 → Automations → New Automation → 粘贴 Prompt → 设置时间 → Test Run → Activate。
适合：工作记录本来就在数字系统里（日历、Notion、邮件），不需要额外整理，每周周报格式固定。
三种方式怎么选
建议： 第一次用先从手动喂料版开始，熟悉之后根据你的记录方式选择文件直投版或直接跳到挂机自动版。

直接接管版——Computer Use 操控 Excel 桌面应用（最强，适合复杂 GUI 和内网文件）

注意事项

场景六：Codex 操控你的电脑——从"帮你想"到"帮你做"

前五个场景，Codex 扮演的角色是"聪明的写手"：你问它，它答你，你来执行。
这个场景，Codex 的角色彻底变了：它直接坐到你的电脑前，替你操作。
这件事是从什么时候开始的
2026 年 4 月 16 日，OpenAI 推出了 Codex 的重大更新，让它能够在 Mac 上看见、点击并在任何应用程序中输入文字——同时你可以继续做其他事情。
OpenAI 承认，在这次更新之前，已经有 50% 的 Codex 用户在用它完成非编程任务。这让 Codex 成为了整合各类生产力功能的自然载体。
这次更新的核心能力，分三层：
Computer Use（手动喂料版）——Codex 能看到你的屏幕，控制鼠标和键盘，操作任何 macOS 应用。
内置浏览器（文件直投版）——Codex 有自己的浏览器，可以打开网页、填写表单、提取信息，不占用你正在用的浏览器。
Automations（挂机自动版）——你设置一个任务、定好时间，Codex 在后台自动运行，不需要你在场。
手动喂料版——替你操作没有 API 的系统
它能做什么
Codex 的 Computer Use 功能，让它能在 macOS 上看到并操作图形界面——包括检查桌面应用、使用浏览器、修改应用设置，以及处理那些没有 API 接口的数据源。
场景三的批量脚本，处理的是本地文件。而很多公司的数据锁在 OA、ERP、内网系统里，没有导出接口——这时候 Computer Use 就是唯一的解法。

Codex 可以按照你设定的计划自动运行任务。它不再等你来发问，而是在预定时间主动执行，完成后把结果呈现给你审阅。

设置 Automations 的操作步骤
  1. 打开 Codex 桌面 App（macOS 或 Windows）
  2. 左侧边栏点击「Automations」→「New Automation」
  3. 给任务起名（比如"每日早间简报"）
  4. 粘贴 Prompt，设置执行频率（每小时 / 每天 / 每周 / 自定义）
  5. 连接所需插件（Gmail、Slack、Google Calendar 等）
  6. 点击「Test Run」先跑一次确认结果
  7. 点击「Activate」，任务开始定期自动运行
一个实用原则：先在普通对话里测试 Prompt，确认结果符合预期之后，再把它转化为定时自动化任务。

平台限制： Computer Use 目前主要在 macOS 上运行，Windows 支持正在跟进。
地区限制： Computer Use 和记忆功能目前在欧洲经济区暂不可用，其他地区逐步开放。国内用户需确认账号访问权限。

审批权始终在你手里： Codex 在你明确授权之前，不会主动发送邮件、提交表单或修改数据。它可以监控、起草、整理，但外部动作由你批准。

理解了底层逻辑，你才能在本文没覆盖到的场景里自己想出好的 Prompt。
OpenAI 官方文档总结了让 Codex 产出高质量结果的四个核心要素：上下文（哪些文件、信息和背景与这个任务相关）、约束（需要遵守什么标准、规范、格式要求）、完成标准（什么情况算做完了）、以及任务本身（你想要什么）。
翻译成普通话就是四个问题：
1. 我是谁，我在什么情况下？（背景）告诉 Codex 你的职业、行业、文档格式。
2. 输入是什么？（素材）把你有的东西全部给它：原始数据、粗稿、聊天记录、要求清单。
3. 输出要什么样？（格式和要求）"帮我整理一下"和"整理成 500 字的文档，结构包括……，语气要……"，差别巨大。
4. 我不要什么？（约束）负向约束同样重要，帮 Codex 避开你最在意的雷区。
最后，把六个场景汇总，方便收藏：
选一个你这周就会遇到的场景，把对应的 Prompt 复制过去，改成你的实际内容，发给 Codex。
大概率，你花在上面的时间，会比你想象的少很多。
