下面是 Claude 的完整产品线，简单说：
  1. Claude "Chat" → 就像 ChatGPT，可能是你唯一知道的那个。
  2. Claude "Project" → 还是 Claude Chat，只不过按项目分开管理。
  3. Claude "Code" → 开发者的革命性工具，写代码快了不止一个档次。
  4. Claude "Cowork" → 类似 Claude Code，但面向我们这些知识工作者（knowledge workers）。
  5. Claude "Skills" → 教 Claude 可重复的工作流，比 Project 更强。
  6. Claude "连接器（Connectors）" → 把 Claude 直接接入 Slack、Google 日历、Gmail 等应用。它能在你日常使用的工具里读、写、执行操作。
  7. Claude "插件（Plugins）" → 跟连接器类似，但得你自己动手搞（你不会想这么干的）。连接器 = 从应用商店下载，插件 = 你自己上传应用。

下面教你怎么建文件夹：

第一步：搭建你的文件夹。

在电脑上为 Cowork 创建一个专用文件夹。
它需要一个干净的、有规划的空间。我的是这样的：
ABOUT ME*→ 一个文件夹，里面放 1/ "关于我"，2/ "反 AI 写作风格"。 PROJECTS → 进行中的工作。你正在做的项目。简报、草稿、该项目的参考资料。每个项目一个子文件夹。 TEMPLATES → 做得足够好、值得复用的成品，作为模板。不是内容本身，而是"完美"的结构，当模板用。 CLAUDE OUTPUTS → Claude Cowork 给你交付成品的地方。
父文件夹：CLAUDE COWORK。里面有 4 个子文件夹（1-2-3-4）。
这样做能保持条理清晰，也限制了 Claude 能看到的范围。Cowork 对你共享的文件夹拥有真实的读写权限。万一出了问题，你要把影响范围控制住。把权限收紧。

第三步：设置全局指令（Global Instructions）。

进入：Settings → Cowork → Edit Global Instructions。
这相当于一个持久化的提示词，Claude Cowork 每次都会先读它。
全局指令（Global Instructions）控制的是 Claude 必须（始终）遵守的行为规范。
粘贴以下内容：
# 全局指令 ## 每次任务开始前 1. 读取 `ABOUT ME/`。任何任务都必须先读完这两个文件。 2. 如果任务与某个项目相关，先读取 `PROJECTS/` 中对应子文件夹的所有内容，再开始工作。 3. 如果任务涉及的内容类型在 `TEMPLATES/` 中有对应模板，先研究该模板的结构。使用结构，不要照搬内容。 ## 文件夹协议 你有三个只读文件夹和一个可写文件夹。 ### 只读 — 绝不在这里创建、编辑或删除任何内容： - `ABOUT ME/` → 我的身份信息和写作规则。 - `TEMPLATES/` → 经过验证的结构，作为模式复用。 - `PROJECTS/` → 我的简报、参考资料和已完成的作品，按项目组织。 ### 可写文件夹 — 你唯一可以交付工作的地方： - `CLAUDE OUTPUTS/` → 你创建的所有内容都放在这里。按项目建子文件夹，结构与 `PROJECTS/` 保持一致。如果子文件夹不存在，自行创建。## 命名规范 你创建的所有文件必须遵循以下格式： `project_content-type_v1.ext` 内容类型：Newsletter, LinkedIn Post, Brief, Deck, Report。 示例： - `How-To-AI_Newsletter_v1.md` - `EasyGen_Deck_v1.pptx` - `GPC_Report_v2.docx` 如果已存在同名文件，递增版本号。 ## 运行规则 - 如果简报不清楚或不完整，使用 `AskUserQuestion` 工具提问。不要用泛泛的内容填补空白。 - 不要过度解释。交付成果就行。除非我主动问，否则省掉那些评论。 - 绝不删除任何地方的文件。
设置一次就行。以后每次都会自动生效。你再也不用重复输入了。

Cowork 的短板（我答应过说实话的）。

它烧额度烧得很快。一次 Cowork 会话消耗的额度，顶得上你平时聊几十次普通对话。如果你用的是 Pro 套餐（$20/月），每天都用的话，一周之内就会感觉到额度紧张。如果 Cowork 成了你的主力工作流，建议升级到 Max 套餐（$100/月）。我把话说在前面，免得你到时候措手不及。
它还是研究预览版（research preview）。Anthropic 自己也明确说了。它会犯错。它会读错文件。有时候明明有更简单的做法，它偏偏绕了个弯。你必须检查它的产出。别不看一眼就把东西直接发给客户。
它需要应用保持打开。关掉 Claude 桌面端，会话就断了。没有手机版。没有网页版。Cowork 只能在 macOS 或 Windows 的桌面端里运行。
它不适合快问快答。想问"法国首都是哪"？用 Chat 就行。Cowork 是干活用的，不是查百科的。它是为多步骤任务设计的。
AI 智能体（Agent）有时候会翻车。遇到复杂任务时，Cowork 会把工作拆成几块，用多个智能体并行处理。大多数时候又快又准。但偶尔有个智能体会跑偏，最终产出里就会有一段跟其他部分画风不一样。盯着点就行。大概 10% 的概率会出现。
Cowork 不是万能的。但它每周都在变强。
而且这才是 Claude Opus 4.6 + Cowork 的水平，我都不敢想 Claude 5、Claude 6 会是什么样……

第 5–10 分钟：搭建你的上下文文件夹。

→ 在电脑上创建一个文件夹叫 "Claude-Cowork"。里面建四个子文件夹：ABOUT ME、TEMPLATES、PROJECTS、CLAUDE OUTPUTS。
→ 在文件夹里创建你的第一个文件：about-me.md。写三样东西：（1）你是做什么工作的。（2）你喜欢什么样的沟通方式。（3）一篇你自己觉得写得不错的文章，贴进去。

第 10–15 分钟：开始你的第一次 Cowork 对话。

→ 在 Cowork 里，点击 Add Folder，选择你的 Claude-Cowork 文件夹。

第 15–20 分钟：安装一个插件。

→ 点击侧边栏的 Customize → Browse plugins（浏览插件）。
→ 选一个适合你工作的。推荐入门款：Productivity（任务和工作流）、Marketing（内容起草）、或 Sales（潜客调研）。
→ 点击安装。开一个新对话，确保插件已激活。
→ 输入 / 看看有哪些斜杠命令（slash commands）可用。试一个。

第 20–30 分钟：搞一个真正的交付物。

→ 给 Cowork 一个你这周工作中真正需要完成的任务。
→ 输入："根据这个文件夹里的文件，帮我做一份 [报告 / PPT / 文档]。先问我问题。"
→ 看它生成一个真实的文件。打开它。编辑它。用起来。
