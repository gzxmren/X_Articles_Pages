我给 Obsidian 做了个AI 创作台：打开笔记，就能让 Claude Code 直接改它
135K
兄弟们，今天给你一个能直接在Obsidian上手使用的 AI 写作工作台。
它可以帮你把Obsidian中的笔记生成公众号文章，小红书甚至是选题。如果你是一名内容创作者，你一定要上手试试。
先说结论：Obsidian 和 Claude Code 结合，最有价值的不是做一个漂亮 Dashboard。
真正有价值的是，你打开一篇笔记，点一下按钮，Claude Code 就能围绕这篇笔记做事。
比如：
  • 把当前笔记改成公众号文章；
  • 把当前素材拆成小红书内容；
  • 从当前草稿里提炼选题；
  • 给当前笔记补摘要、标签和双链；
  • 需要深度修改时，直接在 Obsidian 底部和 Claude Code 继续聊。
这才是我觉得普通创作者真正能用起来的工作台。
不是多一个聊天窗口。
而是让 AI 进入你的内容现场。

01. 先准备三个东西

这套工作台需要三个基础组件。
第一，Obsidian
Obsidian 是本地 Markdown 笔记软件。你的笔记不是锁在某个云文档里，而是一个个 .md 文件，放在你的电脑文件夹里。
这点很关键。
因为 Claude Code 能直接读写本地文件。
下载地址：
https://obsidian.md/download
下载安装后，新建或打开一个 vault。
你可以把 vault 理解成一个笔记库文件夹。
比如：
text
/Users/can/Obsidian
第二，Claude Code
Claude Code 是命令行里的 AI 编程/文件操作助手。
它不只是能写代码，也能读取、整理、改写 Markdown 文件。
官方安装文档：
https://code.claude.com/docs/en/setup
macOS / Linux / WSL 可以用官方安装命令：
bash
curl -fsSL https://claude.ai/install.sh | bash
如果你更习惯 Homebrew，也可以用：
bash
brew install --cask claude-code
安装后，在终端里验证：
bash
claude --help
只要这个命令能正常输出帮助信息，就说明 Claude Code CLI 已经可用。
第三，Obsidian Terminal 插件
Terminal 插件的作用，是把真正的终端嵌进 Obsidian。
这样你不用在 Obsidian 和系统终端之间来回切。
插件地址：
https://github.com/polyipseity/obsidian-terminal
安装方式：
  1. 打开 Obsidian；
  2. 进入 Settings -> Community plugins；
  3. 关闭 Restricted mode；
  4. 点击 Browse；
  5. 搜索 Terminal；
  6. 安装并启用。
Obsidian 官方的社区插件安装说明在这里：
https://obsidian.md/help/community-plugins
启用 Terminal 后，把终端面板拖到底部。
然后在里面进入你的 vault：
bash
cd "/你的/Obsidian/Vault/路径"
claude
这一步的意义是：你可以在 Obsidian 里保留一个完整 Claude Code 会话。
按钮解决固定动作。
Terminal 解决连续对话。

02. 再安装当前笔记操作台

接下来安装我做的这个插件。
它不是 Obsidian 官方插件市场里的插件，而是一个开源 starter kit。它不是 Obsidian 官方插件市场里的插件，而是一个开源 starter kit。
项目地址：
https://github.com/SIXIANGGUO/cc-note-ops
下载压缩包：
https://github.com/SIXIANGGUO/cc-note-ops/releases/download/v0.5.6/cc-note-ops-v0.5.6.zip
拿到压缩包后，先解压：
bash
unzip cc-note-ops-v0.5.6.zip
这个安装包在文章的末尾，我已经开源了，大家下载就可以。
进入目录：
bash
cd cc-note-ops-v0.5.6
运行安装脚本，把它装进你的 Obsidian vault：
bash
bash scripts/install.sh "/path/to/your/ObsidianVault"
把 "/path/to/your/ObsidianVault" 换成你的 Obsidian 笔记库路径。
比如你的 vault 是：
text
/Users/can/Obsidian
那命令就是：
bash
bash scripts/install.sh "/Users/can/Obsidian"
安装脚本会自动复制三类东西。
注意，我没有把一堆工程目录铺到你的文件区。
日常你只需要看到：
text
控制中心/
脚本放在隐藏目录里，避免左侧文件树太乱。

03. 启用插件

安装完成后，重启 Obsidian。
然后打开：
text
Settings -> Community plugins
找到并启用：
text
CC Command Center
启用后，左侧 ribbon 会多一个入口。
打开任意一篇 Markdown 笔记，再点击这个入口。
你会看到一个“当前笔记操作台”。
如果你没有先打开 Markdown 文件，它会提示你先打开一篇笔记。

04. 怎么使用？

使用顺序很简单。
第一步，打开一篇笔记。
它可以是一段素材、一篇草稿、一份口播稿、一篇读书笔记，或者任何 Markdown 文件。
第二步，打开当前笔记操作台。
面板会自动识别当前文件，并显示：
  • 文件名；
  • 文件路径；
  • 字数；
  • 行数；
  • 修改时间；
  • 内容预览。
第三步，点按钮。
当前版本内置了 6 个动作。
建议第一次使用时，先点不会改原文的按钮。
比如：
text
提炼选题
如果 Claude Code 正常，它会生成：
text
控制中心/运行结果/当前笔记/你的笔记名-选题池.md
再试：
text
公众号改写
它会生成：
text
控制中心/运行结果/当前笔记/你的笔记名-公众号文章.md
这两个动作都不会修改原文。
确认没问题后，再试更敏感的动作：
text
备份后润色原文
这时候插件会先把当前笔记复制到：
text
控制中心/备份/
然后再让 Claude Code 修改原文件。
如果结果不满意，就去备份目录找回旧版本。
六大功能，涵盖了 AI 创作

05. Terminal 插件怎么配合？

当前笔记操作台适合固定动作。
Terminal 适合连续对话。
比如你点了“公众号改写”，生成了一篇文章。
但你觉得开头还不够锋利。
这时你不用重新设计按钮。
直接到底部 Terminal 里对 Claude Code 说：
text
打开刚生成的公众号文章，把前三段改得更有冲突感，但不要夸张。
或者你想先讨论，不想立刻改文件：
text
先别改文件，先给我 3 个改写方向，每个方向说明适合什么读者。
这就是两者的分工：
所以这套东西不是让插件取代终端。
而是把常用动作做成按钮，把复杂动作留给终端。

06. 最终能达到什么效果？

装好以后，你会得到一个很清楚的工作流。
text
打开一篇笔记
↓
点一个按钮
↓
Claude Code 读取当前笔记
↓
生成公众号 / 小红书 / 选题 / 摘要
↓
结果写回 Obsidian
↓
需要细改时，用 Terminal 继续接管
小红书的话，可以一键复制生图提示词，方便你粘贴到生图模型
这对内容创作者很实用。
因为你的输入不再是一段复制粘贴的文字，而是一篇真实存在的 Obsidian 笔记。
Claude Code 处理的也不只是“这段话”，而是一个本地文件。
它知道文件路径。
它知道结果要保存到哪里。
它也可以遵守你 vault 里的 CLAUDE.md 写作规则。
这就比普通 AI 写作插件更像一个本地内容助理。
不是“帮你补一句话”。
而是“围绕这篇笔记，把它变成下一个可交付内容”。

07. 为什么要做本土化？

很多海外 AI 工作台会放 YouTube、Instagram、TikTok、Google Calendar 这类东西。
但中文创作者真正需要的不一定是这些。
你更可能关心：
  • 这篇笔记能不能变成公众号；
  • 能不能拆成小红书；
  • 能不能做视频号脚本；
  • 能不能沉淀成知识库条目；
  • 能不能下次继续复用。
所以这个插件默认按钮不是海外平台那一套。
它默认服务的是中文内容创作者。
先让一篇笔记变成多个可发布内容。
再让每次输出回到 Obsidian，变成长期资产。

08. 如何卸载？

这类插件会创建隐藏目录，所以卸载必须讲清楚。
如果你只想删除插件和隐藏脚本，但保留已经生成的内容，运行：
bash
bash scripts/uninstall.sh "/你的/Obsidian/Vault/路径"
如果你连 控制中心/ 里的生成结果和备份也不要了，运行：
bash
bash scripts/uninstall.sh "/你的/Obsidian/Vault/路径" --remove-content
默认卸载会删除：
text
.obsidian/plugins/cc-command-center/
.cc-command-center/
加 --remove-content 后，才会删除：
text
控制中心/
工具可以复杂。
卸载必须简单。

09. 最后

这套东西最重要的，不是 UI。
UI 只是入口。
真正重要的是，你能不能把每天重复发生的内容动作，固化成一个按钮。
打开笔记。
点一下。
生成结果。
必要时在 Terminal 里接着问 Claude Code。
这才是 Obsidian 和 Claude Code 放在一起时，最值得普通创作者学会的用法。
别把 AI 工作台做成展厅。
要把它做成厨房。
