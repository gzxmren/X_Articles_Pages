1. 基础安装

务必使用 64 位 Ubuntu 系统，否则可能需要自行构建或安装复杂依赖
使用官方一键安装脚本，自动配置 Node.js 环境及 Clawdbot 核心组件：
bash
curl -fsSL https://molt.bot/install.sh | bash
本体安装有无数的教程，请自行解决并确保 Clawdbot 正常运行
可以参考
官方文档

2. 配置 Telegram 机器人

注意：如果安装 Clawdbot 初始化的时候已经配置过 telegram 可以跳过此步
  1. 获取 Token：在 Telegram 中私聊
    @BotFather
    ，使用 /newbot 指令创建机器人并保存生成的 API Token。
  2. 写入配置： 打开 ~/.clawdbot/clawdbot.json 将 Token 写入 Clawdbot 配置：
json
{
  "gateway": {
    "channels": {
      "telegram": {
        "enabled": true,
        "botToken": "YOUR_BOT_TOKEN"
      }
    }
  }
}
  1. 重启服务：配置完成后，系统会自动重启 Gateway 以应用更改。

3. 开启 Telegram 实时流式输出 (Stream Mode)

为了让机器人回复时具有逐字/逐句跳出的“打字机”效果，需要同时配置机器人配置和 BotFather 设置。
Step 1: 修改 Clawdbot 配置
启用 streamMode: "partial"：
json
{
  "gateway": {
    "channels": {
      "telegram": {
        "enabled": true,
        "botToken": "YOUR_BOT_TOKEN",
        "streamMode": "partial"
      }
    }
  }
}
Step 2: BotFather 关键设置
流式输出依赖于 Telegram 的特定消息处理模式，建议通过
@BotFather
的 Web App/小程序界面进行设置：
  1. 进入 BotFather 小程序 管理界面。
  2. 选中你的机器人 -> 点击 Bot Settings。
  3. 找到 Threaded Mode 开关并将其打开。
注意：请直接在小程序界面操作，而不是在聊天框发送对话指令。
配置完成后，建议运行 clawdbot gateway restart 重启网关以确保生效。

4. 安装与配置 Google Chrome

为了使机器人具备网页浏览和自动化能力，我们需要安装原生的 Google Chrome。
安装 Chrome 浏览器
bash
wget https://dl.google.com/linux/direct/google-chrome-stable_current_amd64.deb
sudo dpkg -i google-chrome-stable_current_amd64.deb

# 如果报错依赖缺失，运行下面这行
sudo apt --fix-broken install -y
在 Clawdbot 中启用浏览器
修改 ~/.clawdbot/clawdbot.json 配置，指定 Chrome 路径并开启控制端口：
json
{
  "browser": {
    "enabled": true,
    "controlUrl": "http://127.0.0.1:18791",
    "executablePath": "/usr/bin/google-chrome-stable",
    "headless": true,
    "noSandbox": true,
    "defaultProfile": "clawd",
    "profiles": {
      "clawd": {
        "cdpPort": 18800,
        "color": "#FF4500"
      }
    }
  }
}

4. 远程登录与调试 (chrome://inspect)

当需要给机器人登录账号（如微信、GitHub）时，可以通过 SSH 隧道转发 Chrome 的调试端口到本地。
  1. 建立 SSH 隧道： 在本地终端执行（假设服务器 IP 为 your_server_ip）：
bash
ssh -L 9222:localhost:18800 root@your_server_ip

# 注意：18800 是 Clawdbot 默认分配给配置文件的 CDP 端口。
本地浏览器访问：在本地 Chrome 地址栏输入：chrome://inspect/
#devices
点击 Configure...，确认包含 localhost:9222。
在下方 "Remote Target" 中你会看到 Clawdbot 正在运行的页面，点击 inspect 即可像操作本地浏览器一样打开任意网站登录你的帐号。

5. 安装 Whisper 语音识别

环境准备
bash
# 安装 ffmpeg (必需)
sudo apt update && sudo apt install ffmpeg -y

# -U 参数确保安装最新版本
pip install -U openai-whisper
Clawdbot 集成配置
在 ~/.clawdbot/clawdbot.json 配置文件中添加音频处理模型
注意，到了这一步你甚至可以把配置在聊天框里丢给 Clawdbot 让他自己配置
bash
{
  "tools": {
    "media": {
      "audio": {
        "enabled": true,
        "models": [
          {
            "type": "cli",
            "command": "/usr/local/bin/whisper",
            "args": [
              "--model",
              "base",
              "--output_format",
              "txt",
              "{{MediaPath}}"
            ],
            "timeoutSeconds": 120
          }
        ]
      }
    }
  }
}
配置完成后，机器人收到语音消息时会自动调用 Whisper 进行转录。

常用管理指令

  • 查看状态：clawdbot gateway status
  • 重启网关：clawdbot gateway restart
  • 查看日志：clawdbot logs
