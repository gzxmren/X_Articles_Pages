The 10 Claude Code Variables You Only Need to Set Once
18K
Claude Code has 50+ environment variables. Most of them are internal or for enterprise setups.
But 10 of them directly change how Claude Code thinks, performs, and costs you money.
Add them once to your terminal config, every session after that is permanently better. Here are all 10 👇

How to work with variables?

Every variable below goes into your terminal config file (~/.zshrc on Mac, ~/.bashrc on Linux).
Or add them to ~/.claude/settings.json if you prefer config files over terminal setup.
Both ways work, full copy-paste blocks at the end.

1. CLAUDE_CODE_DEFAULT_EFFORT

The single most impactful variable. Anthropic lowered the default thinking effort from high to medium in March 2026. An AMD executive measured a 67% drop in reasoning quality across 6,852 sessions.
bash
export CLAUDE_CODE_DEFAULT_EFFORT=high
Forces Claude to think harder on every turn. The difference is visible immediately: longer analysis, better code, fewer bugs. Boris Cherny confirmed this on Hacker News.
Options: low, medium, high, max

2. CLAUDE_CODE_DISABLE_ADAPTIVE_THINKING

Since February 2026, Claude autonomously decides how much to reason per turn. On "easy" tasks it barely thinks, even when thinking would catch bugs you'd miss.
bash
export CLAUDE_CODE_DISABLE_ADAPTIVE_THINKING=1
Forces a fixed reasoning budget every turn. No more Claude deciding your refactoring task is "simple enough" to skip deep analysis.

3. CLAUDE_CODE_SUBAGENT_MODEL

By default, subagents use the same model as your main session. If you're on Opus, every subagent runs Opus.
That's 5x the cost per agent with often no quality benefit.
bash
export CLAUDE_CODE_SUBAGENT_MODEL="claude-sonnet-4-5-20250929"
Your main session runs Opus for complex reasoning. Subagents handle focused tasks on Sonnet at 1/5 the cost.
Same results, fraction of the token spend.

4. ANTHROPIC_MODEL

Override the default model without using /model every session.
bash
export ANTHROPIC_MODEL="claude-sonnet-4-5-20250929"
Set it to Sonnet for daily work. Switch to Opus with /model when you genuinely need deep reasoning. Saves you from accidentally burning Opus tokens on simple tasks.

5. CLAUDE_CODE_MAX_OUTPUT_TOKENS

Claude's default output can cut off mid-response on complex tasks. This caps (or extends) the output length per response.
bash
export CLAUDE_CODE_MAX_OUTPUT_TOKENS=16384
Higher value for tasks that need long outputs (code generation, documentation). Lower value to keep responses tight and save tokens.

6. CLAUDE_CODE_NO_FLICKER

If you use Claude Code in VS Code's integrated terminal, tmux, or iTerm2, you've seen the screen flickering during long outputs.
Most people just live with it.
bash
export CLAUDE_CODE_NO_FLICKER=1
Enables the experimental no-flicker renderer. Eliminates terminal flicker and adds mouse event support. Night and day difference in VS Code.

7. CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC

By default Claude Code sends telemetry, error reports, feedback surveys, and usage data to Anthropic.
If you work with sensitive codebases or just want a cleaner setup:
bash
export CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC=1
Equivalent to setting DISABLE_TELEMETRY=1, DISABLE_ERROR_REPORTING=1, DISABLE_FEEDBACK_COMMAND=1, and CLAUDE_CODE_DISABLE_FEEDBACK_SURVEY=1 all at once.
One variable instead of four.

8. CLAUDE_CODE_SHELL

Claude Code auto-detects your shell, but sometimes gets it wrong. Especially on machines with multiple shells installed or non-standard setups.
bash
export CLAUDE_CODE_SHELL="/bin/zsh"
Forces Claude to use the correct shell. Prevents the "why is this bash command failing" debugging sessions that turn out to be a shell mismatch.

9. CLAUDE_CODE_HIDE_CWD

Every time Claude Code starts, it shows your full working directory path in the startup banner.
If you're screensharing, streaming, or recording tutorials, this can leak your username and folder structure.
bash
export CLAUDE_CODE_HIDE_CWD=1
Hides the working directory from the startup banner. Small thing but useful for content creators and anyone doing live demos.

10. CLAUDE_CODE_PLAN_MODE_REQUIRED

Forces Claude to plan before executing on complex tasks. Without this, Claude sometimes jumps straight into coding, makes mistakes, then backtracks.
bash
export CLAUDE_CODE_PLAN_MODE_REQUIRED=1
Boris Cherny said plan mode doubles or triples success rate on complex tasks.
This variable enforces it by default instead of remembering to ask "plan first" every time.

The full .zshrc block (copy-paste ready)

Add this to the bottom of your ~/.zshrc (or ~/.bashrc):
bash
# Claude Code configuration
export CLAUDE_CODE_DEFAULT_EFFORT=high
export CLAUDE_CODE_DISABLE_ADAPTIVE_THINKING=1
export CLAUDE_CODE_SUBAGENT_MODEL="claude-sonnet-4-5-20250929"
export CLAUDE_CODE_MAX_OUTPUT_TOKENS=16384
export CLAUDE_CODE_NO_FLICKER=1
export CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC=1
export CLAUDE_CODE_SHELL="/bin/zsh"
export CLAUDE_CODE_PLAN_MODE_REQUIRED=1
Then reload:
bash
source ~/.zshrc
8 lines. Every Claude Code session after this is faster, smarter, and cheaper.

Alternative: set them in settings.json

If you prefer config files over shell variables, add them to ~/.claude/settings.json:
json
{
  "env": {
    "CLAUDE_CODE_DEFAULT_EFFORT": "high",
    "CLAUDE_CODE_DISABLE_ADAPTIVE_THINKING": "1",
    "CLAUDE_CODE_SUBAGENT_MODEL": "claude-sonnet-4-5-20250929",
    "CLAUDE_CODE_MAX_OUTPUT_TOKENS": "16384",
    "CLAUDE_CODE_NO_FLICKER": "1",
    "CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC": "1",
    "CLAUDE_CODE_PLAN_MODE_REQUIRED": "1"
  }
}
Same result, different location. Shell variables take precedence over settings.json.
